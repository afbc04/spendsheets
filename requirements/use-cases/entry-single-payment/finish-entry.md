# Summary

- **Title:** Finish Single Payment Entry
- **Description:** Administrators can finish existing single payment entries
- **Pre-condition:** User must be Administrator & entry must exists and be in draft mode
- **Pos-condition:** Entry's information is updated

# Flow

### Normal Flow

1. User initiates the finish single payment entry process.
2. User selects the entry
3. System marks entry has DONE
4. System fill some information, such as:
    - Current date as date of finish entry
    - Update last change date
