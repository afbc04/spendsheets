# Summary

- **Title:** Entry Details
- **Description:** The system gives details related to an entry
- **Pre-condition:** Entry must exists & User can read
- **Pos-condition:** Entry's details are shown

# Flow

### Normal Flow

1. User initiates the entry details process.
2. System provides informations about this entry, such as:
    - ID
    - ID of Category
    - ID of Monthly Service
    - Is it visible?
    - Type
    - Dates of:
        - Last change
        - Creation
        - Finish
    - Was generated by system?
    - Came from saving?
    - Is it a draft?
    - Money added or removed
    - Description
    - Notes, indicating:
        - ID of note
        - Note
        - Date
    - Status
    - List of tags
    - Scheduled & Real Due Date
    - List of movements, indicating:
        - ID of line
        - Money added or removed
        - Date
        - Comment
    - Money left until its paid
    - Money paid

### Alternative Flow (2) [Entry is deleted/cancelled/ignored]

2.1. System also provides more information, such as:
    - Date of deletion/cancellation/ignoration
