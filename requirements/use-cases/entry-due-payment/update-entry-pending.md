# Summary

- **Title:** Update Due Payment Entry as PENDING
- **Description:** Administrators can edit existing due payment entries' status
- **Pre-condition:** User must be Administrator & entry must exists and be in DRAFT mode
- **Pos-condition:** Entry's information is updated

# Flow

### Normal Flow

1. User initiates the update due payment entry status to PENDING process.
2. User selects the entry
3. System updates the due payment entry, updating:
    - Status to PENDING or SCHEDULED
    - Update last change date
