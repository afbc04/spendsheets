# Main Pages

### Entries Page

- Be able to list entries, seeing:
    - ID
    - When was created
    - Type
    - Total Money
    - Category
    - Status
- Be able to see due payments and not completed loans
- Be able to create a new entry
- Be able to see more information about an entry
- Be able to see categories
- Be able to see tags
- Be able to see services
- Entries can be order by:
    - Last chance date
    - Total money
    - ID
- Trip can be filter by:
    - Type
    - Category
    - Tags
    - Status

### Entry Details

- Be able to update, cancel or delete an entry
- Be able to see information, such as:
    - ID
    - Category related to entry
    - Lists of tags
    - Type
    - List of movements
    - Begin and Due Date
    - ID of service related to entry
    - Comments
    - Status
    - Money added
    - Money removed

# Validations

- **User level** can be:
    - Traveller
    - Driver
    - Administrator
- **Visibility of user** if is private:
    - Some details are shown
    - Trips, as traveller, are not shown
- **Dates** must follow these formats:
    - `yyyy-mm-dd` _year-month-day_
    - `yyyy-MM-dd hh:mm:ss` _year-month-day hour:minute:second_
    - _Only digits allowed_
- **Email** should have this format:
    - `<name>@<domain>.<TLD>`
    - Name length > 0
    - Domain length > 0
    - TLD >=2 
- **Sex** can have these options:
    - Male
    - Female
    - Non specified
- **Country Code** abbreviation must have 3 letters and follow ISO3:
    - Country code is shown in upper case
    - Country code provided is insensitive letter case
- **Boolean values** are:
    - **False**:
        - false
        - 0
        - ""
    - **True**:
        - true
        - 1
- **Bus Pass ID** must have 3 letters:
    - ID is shown in upper case
    - ID provided is insensitive letter case
- **Locality Level** can be:
    - 1 (nearest)
    - 2
    - 3 (furthest)
- **Bus ID** must have 5 letters:
    - ID is shown in upper case
    - ID provided is insensitive letter case
- **Level of bus** can be these:
    - Standard _(Regular)_
    - Express _(Fast)_
    - Luxury _(Has confort)_
- **Visibility of bus**: 
    - If is private, no user _(travellers)_ can book trips
- **Inactive drivers** cannot create trips
- **Trip status** can be:
    - Scheduled
    - Ongoing
    - Completed
    - Cancelled
- **Trip rating** can be:
    - 1
    - 2
    - 3
    - 4
    - 5
    - Not rated
- Trip mean rating should not use not ratings
- Double numbers should be rounded in the third decimal number
